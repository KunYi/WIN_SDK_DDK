Getting Started

The Microsoft® Windows® Driver Development Kit (DDK) for Windows XP Service Pack 1 provides a development environment for the creation of Windows XP Service Pack 1 drivers. The DDK provides additional driver-specific headers, libraries, sources, tools, and documentation that are used to develop drivers for Windows XP Service Pack 1.

Introduction

This document presents an overview of the DDK and provides essential installation information for this driver development kit. Before installing the DDK, read through this document to determine whether or not to install optional DDK components and how to set up your systems for a trouble-free installation. This document contains the following topics:

  • Installing the DDK
  • Installing the Debuggers
  • Verifying DDK Installation
  • Compiling Your Driver for Win64
  • Information on the Microsoft Web Site
  • Late-Breaking News
Please Read The Release Notes

There have been a number of changes to the DDK. Some of these changes may affect driver compatibility among versions of Windows. As a result, after installing the DDK, it is more important than usual to read the release notes. For your convenience, the release notes may be accessed from the Start menu (Start… Development Kits… Windows DDK… Help… Release Notes).

Installing the DDK

To install the Microsoft® Windows® Driver Development Kit for Windows XP Service Pack 1, you must be able to access a CD-ROM drive from Windows, either locally or through a network.

System Requirements

To use this DDK on your driver-development machine, you will need:

  • Microsoft® Windows XP Service Pack 1, Windows 2000, Windows® 98, Windows® 98 SE, or Windows® ME. If installing on Windows 98 Gold, IE 5.x must be installed before the setup program can be run. Installation on Windows NT® 4.0 is not supported.
  • CD-ROM drive or Internet access.
  • At least 64MB RAM. 128MB of RAM or more is recommended.
  • Up to 400MB hard-disk space for a full installation. Significantly more space may be required to compile all samples; up to 850MB may be required.

To use this DDK on your driver-testing machine, you will need:

  • Microsoft Windows XP Service Pack 1.
  • At least 128MB RAM if you are running a free or retail build of Windows XP Service Pack 1, 256MB RAM if running a checked build. In some cases, checked builds may require 128MB of RAM, depending on the particular drivers loaded on the machine.
  • A second machine that is capable of running Windows 2000 or Windows XP Service Pack 1 for kernel debugging of your under-development drivers. This machine can be your driver-development system if desired.

DDK Installation

Note:    You must have Administrator privileges to correctly install the DDK.

Note:    When installing the Windows DDK, you must choose a path with no embedded spaces. Particular DDK buld components may fail to function correctly when installed in paths with embedded spaces.

Note:    Do not install this DDK over previous versions of this or any other DDK. Improper installation of the DDK can cause the use of old and incorrect files, which will lead to unreliable results in any drivers you develop.

Note:  The installation path and icon placement have been changed. The default installation path is now x:\WinDDK\Kit <BldNum>. The icons are now placed at Programs\Development Kits\DDK Kit <BldNum>.

Perform the following steps:

  1. Execute the setup.exe file from the CD-ROM.
  2. If installing on a Windows 9x, Windows NT, or Windows 2000 computer, answer 'Run the program from it's current location' to the question,which asks "What would you like to do with this file?" OR If installing on a Windows XP computer, answer 'Open' to the question, which asks "Would you like to open the file or save it to your computer?"
  3. If a security warning pop-up appears, which indicates that an 'Authenticode signature not found' error, then click the 'Yes' button
  4. Click the Next button to install the Windows DDK.
  5. Read and accept the Software License Agreement.
  6. Click the Next button to proceed.
  7. Select the installation path (it is suggested to use the path that is displayed).
  8. Click the Next button to proceed.
  9. Choose the components that you would like to install.
  10. Click the Next button to proceed.
  11. Confirm that you want to install the DDK by verifying that the installation path is correct.
  12. Click the Next button to install the kit.

The setenv.bat batch file sets the necessary environment variables to build free or checked driver versions. To run setenv.bat, start either a free or checked console window from the Programs/Development Kits/DDK Kit <BldNum>/Build Environments menu. Drivers can then be built by running the BUILD utility. Invoking build -cZ at the command-line prompt compiles and links all drivers in the source tree of the current subdirectory.

Note:    The build environment is set from the icons for a free or checked build. If the build environment is not created correctly, you can manually set the build environment by going to the %NTDDK%\bin directory and typing setenv.bat DDK_DIRECTORY free/checked, where DDK_DIRECTORY is the full path of the location of the Windows DDK.

Note:    On machines with large hard disk sectors such as "FAT 16" file systems, the setup program may miscalculate the amount of space needed to attempt an installation even though the space is not available.

Note:    There are a number of functions exported in ntoskrnl.lib in Windows XP Service Pack 1 that were not in Windows 2000. When building binaries compatible for both Windows XP Service Pack 1 and Windows 2000 operating systems, one option is if the functionality is expected in another .lib (i.e. libcntpr.lib), list it in the lib path prior to ntoskrnl.lib.

Installing the Debuggers

The Debugging Tools for Windows package features WinDbg, a powerful debugger with a graphical interface that can debug both user-mode and kernel-mode code. This package also includes the debuggers KD, NTSD, and CDB, and many additional tools. The documentation in this package describes the use of these debuggers and includes tips for user-mode and kernel-mode debugging.

Debugging Tools for Windows is available in 32-bit and 64-bit versions. The 32-bit version is appropriate for most users. However, if you are planning on debugging an application on an IA-64 based processor, you should select the 64-bit version. The 64-bit version can be installed only on 64-bit versions of Windows.

These debugging tools require approximately 21 MB of hard disk space.

To obtain the most current version of Debugging Tools for Windows, see  Microsoft Debugging Tools.

If you cannot install Debugging Tools for Windows from this internet site, then use one of the following links:

  1. Install 32-bit Debugging Tools for Windows (Required for most customers)
  2. Install 64-bit Debugging Tools for Windows

Symbols

The new linker strips all debug information from the SYS file and moves the data into a PDB file. The PDB file should be copied to the symbols directory for debugging. Copying the SYS file will not provide debugging information.



Verifying DDK Installation

Start either the free or checked build environment by choosing the appropriate icon from the Windows® DDK menu.

Verify your installation of the Windows DDK by running build -cZ from the %NTDDK%\src directory to build the complete set of installed driver sources. This can take more than 30 minutes to complete, depending on which DDK components are installed.

Compiling Your Driver for Win64

Also included in this release of the DDK is a complete 64-bit build environment for the upcoming Microsoft® Win64™ systems.

To compile your driver with the 64-bit compiler, choose the Checked 64 Bit Build Environment or Free 64 Bit Build Environment icon in Programs\Development Kits\Kit <BldNum>\Build Environment. This will open the correct environment for the 64-bit compiler.

WdmVNIC NDIS Miniport Driver

[This is preliminary documentation and subject to change.]

Summary

This directory contains NDIS Miniport Driver project files for the WdmVNIC NDIS_WDM sample driver.

WMI Support in NDIS Miniport Driver

WdmVNIC sample NDIS Miniport Driver shows how WMI support can be added to NDIS Miniport drivers.

1.       Create a MOF file with WMI class descriptions.

2.       Add this file to your miniport driver project and to SOURCES file.

3.       In project settings dialog add custom build step for the MOF file. (“mofcomp   -WMI -B:$(TargetName).bmf $(InputPath)”)

4.       If you need a header file with GUID declarations add one more custom build step to generate a header file: ($(BaseDir)\bin\x86\wmimofck -h$(TargetName)_wmi.h $(TargetName).bmf).

5.       Add generated BMF file to the miniport resource file as NdisMofResource type.

6.       Create NDIS_GUID Array with WMI guid to custom oid mapping

7.       Handle OID_GEN_SUPPORTED_GUIDS oid.

8.       Add custom oid handlers.

9.       Use either WMI Object Browser (part of WMI SDK from Microsoft) or write your own application to access wmi objects.

Building the Driver

First you must build KNDISWDM in DriverWorks. Open the NdisWdm workspace in DriverWorks, and select BatchBuild under the Build menu. Select both the checked and free configurations and hit the Rebuild All button.

Open WdmVNIC.dsp in MSVC 6.0 IDE, select the “Debug” configuration and build it. The build creates files WdmVNIC.sys and WdmVNIC.nms. You will need DriverWorks, DriverNetworks, and the Microsoft Windows DDK installed to successfully build the driver.

Installing and Running the Driver

First, you should edit the .INF file to add a HW_ID for a PNP device that you have handy, such as some USB device. This device will be used simply to allow the driver to enumerate and install. The driver simulates IRP traffic to and from the lower device and will not actually send IRPs to the device whose ID you supply.

Once the .INF file is modified, the device can be installed and started simply by connecting the PNP device to the system, which will detect it and run you through the normal PNP device installation procedure. You can use the Monitor application to view messages produced by the device and you can load and set breakpoints using SoftICE to explore the functionality of the driver.

Tools

BoundsChecker for Drivers and SoftICE can be used to test some of the features of this driver.

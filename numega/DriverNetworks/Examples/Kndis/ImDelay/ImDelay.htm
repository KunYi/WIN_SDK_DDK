ImDelay NDIS Filter Driver (NDIS 5.0, 5.1)

[This is preliminary documentation and subject to change.]

Summary

This directory contains NDIS Intermediate Driver project files for NDIS 5 Filter driver, ImDelay. ImDelay implements a fixed delay (10 msec) for all packets coming in and out. The samples demonstrates the use of packet lists (e.g., KNdisInterlockedPacketList), private packet pools (e.g., KNdisFilterPacketPool), packet recycling techniques and timers (e.g., KNdisTimer) used with the DriverNetworks NDIS Filter framework.

The ImDelay driver is functionally similar to the Passhtru sample in the Windows XP DDK. Instead of forwarding the packets up and down immediately, however, ImDelay queues the packets internally. The driver runs a periodical 10 msec timer, which processes the queues on a regular basis and forwards the packets up and down the stack accordingly.

For NDIS 5.1 Configurations, the driver demonstrates the use of packet cancel handler, OnCancelSendPackets().

Building the Driver

Open ImDelay.dsp in MSVC 6.0 IDE, select the “Debug” configuration, and build it. The build creates files ImDelay.sys and ImDelay.nms. You will need DriverNetworks and the Microsoft Windows DDK installed to successfully build the driver.

Installing the Driver

In Windows Network Connection Folder, right click on Local Area Connection and select Properties. In the pop-up dialog box, click Install... and then Service. Navigate to the folder where ImDelay.sys, netFilter.inf, and netFilterMP.inf files are located. Click on ImDelay. This will install the driver. Click OK to dismiss the Properties dialog. This will start the driver. No reboot is necessary.

Tools

BoundsChecker for Drivers and SoftICE can be used to test some of the features of this driver. Use the Monitor.exe application to watch the traces produced by the driver when it sends and receives packets.

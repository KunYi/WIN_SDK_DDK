Sample I1394 NDIS Miniport Adapter Driver

[This is preliminary documentation and subject to change.]

Summary

The I1394 driver allows two computers to be networked together using one I1394 cable. Once installed, the driver allows each I1394 port to behave as an Ethernet NIC adapter. The driver is NDIS 5.0 compatible and supports Plug ’n Play on Win2K and WinXP platforms.

This driver implements an NDIS upper edge and a WDM Miniport lower edge. It uses the DriverNetworks class K1394LowerDevice to manage the hardware.

Driver and Networking Setup

Use the Device manager to configure a MAC address for each adapter. To do this, open the device manager and expand the Network Adapter node. This will list all the NIC adapters currently installed in your system. Right-button select the I1394 Firewire Network Adapter # and then click on properties. When the property page appears, select the Advanced tab. The tab shows three configurable items:

·         MAC setting (“Network Address (MAC)”)

·         “Sample Parameter”

·         Transmit IRP pool size setting (“TxIrpPoolSize”)

The “Sample Parameter” was placed here for demonstration purposes, and has no effect on the driver. You may wish to use “001122334455” as the MAC value on one PC and “001122334466” as the MAC value for the second.

Next, for a proper TCP/IP installation you must provide an IP address for each adapter. On each machine, right-button-click “My Network Places” and select the properties option. Right-Button-click on the I1394 adapter by name and select the Properties option. The LAN Connection property page will appear.

Note:  If your system does not have “Client For Microsoft Networks” and “Internet Protocol (TCP/IP)” available, you must install them before continuing.

Make sure that “Client For Microsoft Networks” and “Internet Protocol (TCP/IP)” are check-marked. If they are not then you must click the appropriate box so they become active.

Select the “Internet Protocol (TCP/IP)” option. Then select the “Properties” tab. A pop-up will appear asking you to set-up the IP address. Select the “Use the following IP address and enter the IP value. Then make the Subnet mask 255.255.255.0. Select OK. Then select OK on the LAN Connection property page.

At this point the system is ready.

Building the Driver

First, build the DriverNetworks libraries. You will need the Microsoft Windows DDK installed to successfully build the library. The Microsoft Visual Studio workspace is located in the “source” folder of the DriverNetworks installation folder. Open DNW.dsw in MSVC 6.0 IDE. Use the “Batch Build” command to build the “NDIS 5 Miniport Checked” and all other available configurations of the KndisLib project. If you are planning to run I1394 driver on Windows Me, you need to select Windows 2000 as a target OS when building KndisLib and KdisWdm libraries.

Next, build the I1394 driver. Open I1394.dsw in MSVC 6.0 IDE, select the NDIS 5 Miniport Checked configuration of the I1394 project and build it. It creates files I1394.sys and I1394.nms.

Note:  Project “OID Handlers” is just a “dummy” project to aid in displaying OID handlers in the class browser of MSVC 6.0 IDE. This project cannot be “built.”

Installing the Driver

It is assumed that the both PCs have a 1394 port. Connect the I1394 cable between both the computers. Once Windows recognizes the connection, it will prompt you to install the driver. Follow the directions provided. On Windows Me and XP Microsoft provides device drivers for the direct link connection. In this case you won’t be prompted for I1394 driver and you would need either manually load I1394 instead of the OS drivers (to do that go through Update Drivers… wizard or use EzDrvInstaller) or if the second machine is also Windows XP or Windows Me you can load a virtual 1394 device driver on it (see DriverWorks\Examples\WDM\vdev1394), which will create an additional config rom entry. In the latter case Windows will prompt you for a driver for this new pseudo device and you can point to I1394.

The I1394.inf configuration file is provided in the “..\I1394P2P” directory.

Tools

BoundsChecker for Drivers and SoftICE can be used to test some of the features of this driver. Specifically, the SoftICE “packet” command is useful for displaying and decoding NDIS packet buffers.

StlFilter NDIS Intermediate Filter Driver

SUMMARY

This directory contains project files for the NDIS Intermediate Filter Driver called “StlFilter”.  The sample demonstrates using the C++ Standard Template Library (STL) with an NDIS driver.  Specifically, the string and map classes are used from the STL.  The C++ runtime environment for STL is setup by the DriverNetworks framework.  The following classes are demonstrated:

KNdisDebugTrace, KNdisIMiniDriver, KNdisFilterBinding, KNdisFilterAdapter, KNdisPacket, KNdisBuffer, KNdisConfig.

BUILDING THE DRIVER

Open StlFilter.dsp in MSVC 6.0 IDE, select the “Debug” configuration and build it. It creates files StlFilter.sys and StlFilter.nms. You will need Microsoft Windows 2000 DDK and Compuware NuMega DriverNetWorks framework installed to successfully build the driver.

INSTALLING THE DRIVER

The driver is installed as a Network Service Component Type. From the Local Area Connection Properties dialog, select the Install... button. Select the Service type of network component to install and click the Add... button. This launches the Select Network Service dialog. Click the Have Disk... button and browse to the netFilter.inf file which is the .inf file for the protocol edge of the intermediate driver. Note: the miniport .inf file (netFilterMP.inf) is not selected for install by the user.

TOOLS

The Compuware NuMega DriverWorkbench and SoftICE can be used to test some of the features of this driver.

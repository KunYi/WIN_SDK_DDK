Tcpecho.sys:  Sample Kernel Mode TCP Server

[This is preliminary documentation and subject to change.]

Summary

This directory contains project files for Tcpecho.sys. This sample demonstrates a simple TDI client written using the DriverNetworks framework. Tcpecho driver listens on the TCP echo port (Port 7), accepts connection requests from remote TCP echo clients, and echoes back whatever messages it receives over an active connection. Up to five simultaneous connections can be maintained.

Building the Driver

Open Tcpecho.dsp in MSVC 6.0 IDE, select the “Debug” configuration and build it. It creates files Tcpecho.sys and Tcpecho.nms. You will need DriverWorks, DriverNetworks, and the Microsoft Windows DDK installed to successfully build the driver.

Tools

Use DriverStudio’s DriverMonitor tool to select and start Tcpecho.sys driver on the target IP-enabled machine. On some other IP-enabled machine, start the telnet application and connect to the IP address/echo port of the target machine. Type and make sure the typed symbols are echoed back. Disconnect. Watch the debug traces on the DriverMonitor window.

UsbWebServer.sys:  Sample Kernel Mode Web Server

[This is preliminary documentation and subject to change.]

Summary

This directory contains project files for UsbWebServer.sys. This sample demonstrates a simple TDI client written using the DriverNetworks framework. The UsbWebServer driver listens on the Http port (TCP Port 80), accepts connection requests from remote IP clients, and responds to HTTP GET requests by sending an HTML page to the client. The driver is also performs I/O with the Cypress Semiconductor USB Thermometer to get the current temperature. This is displayed on the HTML page. In addition, the driver opens a Temp.htm file as a 'template' for the response. This file can be edited to display any HTML. The driver processes XML tags <TEMPERATURE> and <WEB_SERVER> located in the Temp.htm file by filling in the Host name and current temperature obtained from the USB thermometer.

Building the Driver

Open UsbWebServer.dsp in MSVC 6.0 IDE, select the “Win32 Windows 2000 Checked” configuration and build it. The build creates files UsbWebServer.sys and UsbWebServer.nms. You will need DriverWorks, DriverNetworks, and the Microsoft Windows DDK installed to successfully build the driver. You must also build the UsbTherm sample from DriverWorks. UsbWebServer.sys is implemented as a USB WDM upper filter driver for the USB Thermometer.

Note:  The driver will not be binary compatible with Windows 9x systems due to differences in the TDI implementation among platforms, but the driver is source-code compatible across WDM platforms.

Installing and Running the Sample

Use the UsbWebServer.inf file included with the sample to install the driver on the 'server' machine. The driver is designed to work with the Cypress Semiconductor USB Thermometer Starter Kit. If there is already a driver installed for the USB Thermometer, first uninstall the driver. Then, plug in the USB Thermometer. Browse to the .inf included with this sample. Browse to UsbTherm.sys and UsbWebServer.sys drivers so they will be loaded by the system.

On some other IP-enabled machine, start the Internet Explorer browser application and connect to the IP address of the server machine. The server returns the HTML page containing the temperature at the web server machine.

Note:  On Windows 98/Me platforms, the driver attempts to create a listening socket when the device is started (during processing of IRP_MJ_PNP/IRP_MN_START_DEVICE). If the underlying TCP/IP driver has not yet loaded, this listening socket request will fail and the UsbWebServer driver will not be loaded. If the socket is successfully created, the driver will successfully start.

On Windows 2000/XP, UsbWebServer uses the TDI PnP framework to receive notification when the underlying TCP/IP driver loads. The driver will not try to initialize its socket before the network is ready.

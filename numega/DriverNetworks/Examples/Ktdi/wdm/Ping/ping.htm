Ping Sample Driver

 

 

This directory contains project files for the ping sample driver. This driver is a WDM TDI client, and is being used to illustrate the classes KRawSocket and KEnumIPInterface.

Operation

The driver sets up an ICMP socket using class KRawSocket so that it may receive any ping packets. It then broadcasts a ping packet using a broadcast address calculated from the local machine’s IP address and subnet mask. The local IP address is obtained with class KEnumIPInterface.  As it receives replies, the driver prints the replying IP addresses to the debugger.

Building the Driver

You will need DriverWorks, DriverNetworks, and the Microsoft Windows DDK installed to successfully build the driver. Before building the sample, make certain that the DriverWorks and DriverNetworks libraries have been built.

Open the workspace ping1.dsw in Visual Studio, and select the build configuration based on the system where the driver will be loaded. The ping driver contains a Win 9x configuration (for Windows 98 and Millennium platforms), a Win2K/WinXP configuration, and an IA64 configuration. Select Build | Rebuild All to build the driver.

Installing the Driver

The driver must be installed through the ‘Add New Hardware’ wizard on the control panel. Select the option to choose a driver from a list, and browse to the folder containing ping1.inf.

KBDCLASS

SUMMARY

The keyboard class sample illustrates the use of DriverWorks classes KUstring and KRegistryKey in a DDK driver.  It also illustrates the steps necessary to convert a DDK driver from C to C++ so that DriverWorks classes can be used. The sample was intended to run on Windows 2000.

BUILDING THE SAMPLE

The files necessary to build the sample include Kbdclass.cpp, Kbdclass.h, newkbd.h, Kbdclass.rc, Kbdlog.rc, the makefile, and the sources. Once built, the sample produces a keyboard class driver (Kbdclass.sys). One binary is produced. Both checked and free builds are available and functional.

The sample can be built from either the command line or from the Visual C++ IDE. 

q             To build from the Visual C++ IDE, open the project, kbdclass.dsw, and select build to build the sample.

q             To build from the command line, run setenv.bat from the Windows 2000 DDK directory.  Then run build.exe in the project directory.

In order to run the sample driver, you must replace kbdclass.sys in winnt\system32\drivers with the new file and then restart the computer to start the sample.  Because of Windows 2000 file protection you will need to boot into safe mode to copy the new driver over the old one.

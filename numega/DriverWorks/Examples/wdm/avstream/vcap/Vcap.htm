Vcap: DriverWorks AVStream Filter-Centric Sample Driver

SUMMARY

Vcap is a filter-centric AVStream video capture driver written using the DriverWorks AVStream C++ Framework.

The driver simulates a video capture source generating 320x240 images in either RGB24 or YUV422 format. 

Functionally, the Vcap driver is similar to and based on the AVSSAMP sample driver provided with Microsoft® Windows® XP DDK.

 

PROGRAMMING TOUR

Vcap driver class, VcapDriver, derives from KsMiniDriver and implements DriverEntry() method, which performs driver's initialization and registration with the kernel AVStream subsystem (KS.SYS).

The VcapDevice class derives from KsDevice and adds no functionality to the base class (i.e. KsDevice could be used instead directly). VcapFilter class derives from KsFilter and implements both static AVStream properties of the filter (sm_Xxx public data members) and event handlers, such as OnCreate() and OnProcess(). VcapPin class derives from KsPin and implements both static AVStream properties of the pin (sm_Xxx public data members) and event handlers, such as OnCreate().

On driver initialization, VcapDriver::DriverEntry() method instantiates the descriptors of Device, Filter and Pin properties based on the compile-time properties (static data propetrties and handler signatuires) of the classes VcapDevice, VcapFilter and VcapPin accordingly. For instance, declaration of VcapFilter::OnProcess() handler in VcapFilter class declaration signifies Vcap is a filter-centric driver. The descriptors are then assembled (using "<<" operator) into a single KS device descriptor, which gets registered with the AVStream subsystem through a call to KsInitializeDriver. This registration defines all the properties required by the AVStream object factories for consecutive  instantiation of the filter and pin objects on the Vcap driver.

The run-time behavior of the driver is managed by the VcapPin::OnSetDeviceState() handler. On the state transition into KSSTATE_RUN state, the driver starts an internal timer (VcapTimer) simulating an interrupt-driven hardware. AVStream queries the filter for frames by calling VcapFilter::OnProcess() method. OnProcess() generates an image into the supplied frame buffer. On each hardware interrupt (VcapTimer::TimerCallback method), Vcap produces a new frame and calls KsFilterAttemptProcessing for AVStream to prompt for further processing.

See comments in all the source files. Also see complete AVStream documentation in the DDK documentation and DriverWorks help AVStream chapter.

 

BUILDING and INSTALLING THE SAMPLE

The sample requires Windows XP DDK.

Start Microsoft Visual Studio 6.0 IDE using DriverStudio SetDDKgo utility. Open driver project file, Vcap.dsw, and build the driver either using the standard IDE build button or DriverStudio DDK Build button.

Alternatively, one can build the sample by typing build -zc in either the standard XP DDK checked or free build environment.

A successful build produces Vcap.sys. To install the driver, right click on Vcap.inf and select Install. When prompted for Vcap.sys, select the built binary.

 

RUNNING THE SAMPLE

Once installation is complete, the driver can be exercised using  Graphedt.exe tool shipped with Windows XP DDK.

Using Graphedt.exe, one can build a graph with Vcap sample acting a video source (the sample appears under "WDM Streaming Capture Devices") and a DirectShow Video Renderer or the VMR default video renderer acting as a sink.

Alternatively, one can locate a graph description file, Vcap.xgr, located in the Vcap project directory and drag it into open Graphedt.exe application window. This would result in the graph being ready to play.

Click the "play" button.  The output produced by the sample is a 320x240 image standard EIA-189-A color bars. By default, Vcap modulates the luminance component (YUV mode) and green component (RGB mode) to "animate" the image.

The sample runs on Windows 2000 and Windows XP only.

1.1  KsCam1394: DriverWorks 1394 Digital Camera Sample Driver

SUMMARY

KsCam1394 sample is a driver for 1394 digital video camera hardware.  The sample was developed using the Orange Micro iBot Firewire Video Conferencing Web Cam.  The driver is a Kernel Streaming video capture driver written using the DriverWorks KStream C++ Framework.  The driver makes use of DriverWorks 1394 WDM C++ Framework.  KStream drivers are legacy.  For new drivers as of Windows XP, Microsoft has provided the new AVStream architecture.  See the AvCam1394 sample for an AVStream implementation.

Functionally, the Cam1394 driver is similar to and based on the 1394DCAM sample driver provided with Microsoft® Windows® XP DDK.

 

PROGRAMMING TOUR

KStream video capture driver:

KsCam1394 driver class, Stream1394Minidriver, derives from KStreamMinidriver and implements DriverEntry() and OnCreateAdapter() methods, which performs driver's initialization and registration with the kernel KStream subsystem (stream.sys).

The Stream1394Adapter class derives from KStreamAdapter and adds 1394 functionality to the base class.  The class delegates 1394 I/O to CamLowerDevice class which derives from K1394LowerDevice. VideoStream class derives from KStream and implements both KStream properties of the driver and event handlers.

See comments in all the source files. Also see complete AVStream documentation in the DDK documentation and DriverWorks help AVStream chapter.  Also, it is useful to read the DirectShow documentation.  This provides useful background for user mode portion of video capture.

 

BUILDING and INSTALLING THE SAMPLE

The sample requires Windows XP DDK.

Start Microsoft Visual Studio IDE. Open driver project file, Cam1394.dsw, and build the driver either using the standard IDE build button or DriverStudio DDK Build button.  Be sure to select a DDK to build with usign the DDK Build Settings dialog from the DriverStudio menu.  

Alternatively, one can build the sample by typing build -zc in either the standard XP DDK checked or free build environment.

A successful build produces KsCam1394.sys. To install the driver, right click on KsCam1394.inf and select Install. When prompted for KsCam1394.sys, select the built binary.

 

RUNNING THE SAMPLE

Once installation is complete, the driver can be exercised using  Graphedt.exe tool shipped with Windows XP DDK.

Using Graphedt.exe, one can build a graph with Cam1394 sample acting a video source (the sample appears under "WDM Streaming Capture Devices") and a DirectShow Video Renderer or the VMR default video renderer acting as a sink.

Click the "play" button. 

The sample runs on Windows 2000 and Windows XP only.

TOOLS

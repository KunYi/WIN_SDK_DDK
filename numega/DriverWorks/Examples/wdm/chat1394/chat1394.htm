Chat1394 Sample

SUMMARY

The Chat1394 sample illustrates how to set up an address range to receive data the 1394 device writes to the host controller’s address space.  The driver must be run on each of two PCs connected through the 1394 bus. 

The sample consists of a driver: chat1394.sys, and an application: chatapp.exe.  The application displays a console for sending and receiving messages from the other machine.  Internally, the application sends read and write requests to the chat device.  Read requests are satisfied only when the other machine, the device, writes to the host controller’s address space.  Write requests are satisfied immediately as the driver writes to the other machine’s address space.

 

BUILDING THE SAMPLE

The driver, chat1394.sys, and application, chatapp.exe, may be built from the command line or from within the Visual C++ IDE.    To build from the Visual Studio IDE, open the workspace file, 1394chat.dsw.  Set the active configuration in the build menu, and then select Build or Rebuild all.  Both free and checked builds are available.

To build from the command line, start an appropriate DDK build environment from the start menu shortcuts installed with the DDK.  Navigate to the chat1394 directory and type “build”.  The environment variable DRIVERWORKS must be set to the DriverWorks directory for the build to be successful.

 

Note: The DriverWorks library, vdw_wdm.lib, must be built before chat1394.sys may be built.

 

RUNNING THE SAMPLE

The driver, chat1394.sys, must be loaded through the device manager.  Often, when two computers are first connected, the other computer will already be listed in the Device Manager under “Other Devices” with the name “Microsoft 1394 PC.”  When the 1394 device is already listed, double click on the device, and, when the properties dialog comes up, select Update driver from the driver tab.  Choose to specify the location of the driver file, and browse to the folder containing chat1394.inf.  The device manager should then load the driver.

If the device is not in the device manager, the driver may be loaded by using the Add New Hardware option in the control panel.

On Windows 2000 and XP, it is also possible to load the driver using the EZDriverInstaller available in the start menu under NuMega DriverStudio | Tools.

Note: To install chat1394.sys on Windows XP, the EZDriverInstaller must be used.  Windows XP automatically loads a network driver when another PC is connected through 1394.  Because the chat1394 driver is not in the same device class as the network driver, Windows will not allow the driver to be installed through the device manager in the normal way.

 

USING CHATAPP.EXE

The chat application is a console with a data entry portion and a display portion.  Messages may be up to 32 characters long.  To send data to the other machine, type in a message, and press enter.  The message will be copied to the display portion of the screen just before it is sent.  Data from the other machine will be displayed as it is received.  Pressing the escape key will exit the application.

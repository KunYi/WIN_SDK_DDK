Sample Multifunctional Driver for a Virtual Device

[This is preliminary documentation and subject to change.]

Summary

The Multifunctional Device Driver example demonstrates how to write a bus driver for a multifunctional device. This example includes a bus driver for a virtual device with read and write functions, a class installer dll, two different drivers for two different functions of a device, and a configuration utility.

Theory of Operation

DDK definition of multifunctional device: “A piece of hardware that supports multiple discrete capabilities, such as audio, mixer, and music, on a single card. This device is sometimes treated as a bus that can be enumerated to locate the devices on that bus”.

From OS point of view multifunction devices can be divided in three classes:

1                     Bus driver enumerate each function as a separate device. Example is multifunctional PCI card, where each function has its own PCI configuration, which allows PCI bus driver to treat it as a separate entity.

2                     Device sits on a standard bus, but bus driver cannot separate multiple functions. Example, two port PCI serial card, which provides only one PCI configuration. In this case PCI bus driver does not understand that device is multifunctional and creates only one PDO. To solve that problem Microsoft provides generic multifunctional driver mf.sys. Its behavior can be controlled through the registry set by INF file. In the above case of two-port PCI serial card, in INF file you would specify that you device is has two different functions and mf.sys would create two separate PDOs.

3                     The rest of multifunctional devices.

Driver for a multifunctional device is essentially a bus driver. It detects each function in device dependent way and creates a separate PDO for each function. PnpManager will detect each PDO as a new device and will install a separate driver for it.

Responsibilities of a bus driver includes:

  • Enumerate each function as a separate logical device
  • Create a device object for each function
  • Maintain a list of all created devices
  • Process Pnp and Power IRPs directed to physical device objects.
  • Manage power state of a device as a whole.
  • Delete a device object when a function is physically removed.

There are two DriverWorks classes design to help in developing a multifunctional device driver: KBus and KPhysicalDevice. KBus class is a collection class of KPhysicalDevice objects. It updates its state on events from both parent FDO and children PDO stacks. KPhysicalDevice class manages IRP processing for physical device objects.

Building the Driver

First, build the DriverWorks libraries. Next build the vbus, vbuscon, vclassinstaller, vread, and vwrite projects using either MSVC or DDK build.exe.

Installing the Driver

First install vbus driver using Add New Hardware dialog, selecting Add New Device, System Devices, and pointing to the vbus.inf file. This will install vclsinst.dll and vbus driver, which creates a vwrite device in its OnStartDevice handler. Windows will prompt you for an inf file and you should point to vwrite.inf. To add or remove a vread device you can use vbuscon utility and when prompted for an inf file, point to vread.inf file.
